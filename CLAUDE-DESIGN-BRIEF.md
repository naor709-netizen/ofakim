# בריף עיצוב — פורטל עיריית אופקים

## 🎯 מה הפרויקט

פורטל ניהול אירועים שנתי לעיריית אופקים. **3 קהלי יעד** עם ממשקים נפרדים:

1. **🔵 עובדי מנהל החינוך** (פלטה כחולה) — יוצרים, עורכים, מוחקים אירועים
2. **🟠 עובדי מחלקת הנוער** (פלטה קורל-כתום) — אותו דבר, מחלקה אחרת
3. **🟢 הורים/תושבים** (פלטה ירוקה-טורקיז) — צופים בלוח, ללא הרשמה

---

## 🛠 הסטאק הטכני

- **Next.js 14** (App Router) + **TypeScript**
- **Supabase** (DB + Auth + Realtime)
- **React 18**
- **CSS inline + CSS variables** (בלי Tailwind כרגע — אפשר להוסיף)
- כיוון: **RTL מלא**, עברית מקצה לקצה
- גופן: **Heebo** או System Font

## 📂 מסכים שצריכים עיצוב

### 1. דף נחיתה ראשי (`/`)
**מה יש**: 3 כרטיסים גדולים — חינוך / נוער / לוח לתושבים. מתחת — קישור לדשבורד מנהלי.

**מטרה**: הרושם הראשון של האתר. צריך להיות חם, מזמין, עירוני אך לא יבש.

### 2. דף התחברות (`/login`)
**מה יש**: כפתורי Google + Microsoft + שדה מייל. אם המשתמש חדש → טופס הרשמה (שם + מחלקה). אם נרשם → מסך "ממתין לאישור".

### 3. גאנט עובדים — שנתי (`/education`, `/youth`)
**מה יש**:
- סרגל עליון בצבע המחלקה
- Hello Bar עם ברכה אישית + סטטיסטיקות
- כפתור "+ אירוע חדש"
- בורר שנת לימודים (תשפ״ה / תשפ״ו / תשפ״ז)
- Toggle שנתי/חודשי + פילטר "המחלקה שלי / כל המחלקות"
- חיפוש
- מקרא צבעים
- **גאנט שנתי**: 12 חודשים × שורה לכל תחום (15 תחומים), שורת חגים אוטומטית
- **גאנט חודשי**: לוח חודש + ניווט בין חודשים
- כפתור צף לבוט AI

### 4. גאנט עובדים — חודשי
לוח של חודש (5×7 ימים), כל יום מציג עד 3 אירועים, חגים מסומנים.

### 5. לוח ציבורי לתושבים (`/luach`)
**מה יש**:
- כותרת ירוקה
- באנר הזמנה לסקר אישי (אם לא מילא)
- Toggle שנתי/חודשי
- חיפוש + פילטרים (גיל, מחלקה)
- גאנט (אותה תצוגה)

### 6. סקר אונבורדינג להורים (`/luach/onboarding`)
4 שלבים: שם משפחה → ילדים+כיתות → תחומי עניין → התראות. פס התקדמות בראש.

### 7. לוח אישי להורה (`/luach/my`)
- ברכה אישית
- פילטר ילד ספציפי
- סטטיסטיקות (סה״כ, החודש, תחומי עניין)
- אירועים מקובצים לפי חודש, כל אירוע ככרטיס עם פס צבע צדדי + כפתורי שיתוף

### 8. דשבורד מנהלי (`/admin`)
- 4 כרטיסי סיכום (סה״כ, חינוך, נוער, חודש עמוס)
- 2 טאבים: סקירה / היסטוריית שינויים
- גרפים: עמודות לפי חודש, פסים לפי תחום, עוגה לחלוקת מחלקות
- ניווט מהיר לעמודי משנה

### 9. ניהול עובדים (`/admin/users`)
- אזור "ממתינים לאישור" (אם יש)
- אזור מנהלי-על
- אזור עובדי חינוך
- אזור עובדי נוער
- כפתור "+ עובד חדש"

### 10. מאגר תשתיות (`/admin/infrastructures`)
רשת של כרטיסים, חיפוש, יצירה/עריכה/מחיקה.

## 🎯 מה אני רוצה לשפר בעיצוב

1. **חם יותר ופחות יבש** — לא טופס ממשלתי משעמם
2. **אנימציות עדינות** — hover, transitions
3. **מובייל מצוין** — האתר חייב להיראות מושלם בנייד
4. **גופן יותר מודרני** — אולי Heebo או Rubik
5. **Hero section יפה יותר** בדף הבית
6. **כרטיסי אירועים** עם הצללה עדינה ופרטים אסתטיים
7. **Empty states** — כשאין נתונים, מסך יפה עם איור ולא רק טקסט
8. **Loading states** — סקלטונים אנימטיביים במקום "טוען..."
9. **Toast notifications** במקום alert()
10. **כפתורי פעולה ראשיים** — בולטים יותר עם gradient או shadow

## 📐 רכיבים שצריכים שדרוג

### כרטיס אירוע (Event Card)
- **כרגע**: רקע לבן, פס צבעוני בצד, מידע בסיסי
- **רוצים**: הצללה עדינה, hover מעודן, אייקונים יפים

### Hello Bar
- **כרגע**: gradient פסטל פשוט
- **רוצים**: יותר אנושי, אולי עם איור/אייקון, אנימציה לסטטיסטיקות

### כפתור "+ אירוע חדש"
- **כרגע**: כפתור פשוט בצבע המחלקה
- **רוצים**: בולט, FAB (Floating Action Button) במובייל, אנימציה ב-hover

### בורר חודשים בגאנט החודשי
- **כרגע**: כפתורים בשורה בגלילה
- **רוצים**: tabs יותר אסתטיים, אולי עם אינדיקטור

### Modal יצירת אירוע
- **כרגע**: modal פשוט, רקע שחור 40%
- **רוצים**: backdrop blur, הופעה חלקה (slide-up), שדות מעוצבים

### תצוגה חודשית
- **כרגע**: רשת פשוטה
- **רוצים**: יותר נשימה, אירועים יפים יותר, חגים בולטים
